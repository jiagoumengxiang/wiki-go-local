# Quick Task 260406-hk: Support empty documents config - use working directory as documents path when config is empty

## Quick Task Plan

**Mode:** quick
**Quick ID:** 260406-hk
**Created:** 2026-04-06

## Task Description

When `documents_dir` in config.yaml is configured as an empty string, use the program's working directory (`os.Getwd()`) as the documents path instead of the ROOT folder.

## Implementation

### Task 1: Add helper function to get effective documents directory path

**Files:** `internal/config/config.go`

**Action:**
- Add a new exported function `GetDocumentsDir(cfg *Config) string` that returns the effective documents directory path
- If `cfg.Wiki.DocumentsDir` is empty string, return `os.Getwd()` (current working directory)
- Otherwise, return `filepath.Join(cfg.Wiki.RootDir, cfg.Wiki.DocumentsDir)`

**Verify:**
- Run tests to verify the function returns correct paths for both scenarios
- Check that empty string uses working directory
- Check that non-empty uses root_dir + documents_dir

**Done:**
- Function implemented and verified
- Documentation added (if needed)

---

## Summary

This is a quick task that adds a helper function to centralize the logic for determining the effective documents directory path. The function will be used throughout the codebase to replace hardcoded `filepath.Join(cfg.Wiki.RootDir, cfg.Wiki.DocumentsDir)` calls.

The implementation is straightforward:
1. Check if `DocumentsDir` is empty
2. If empty, return current working directory
3. Otherwise, return root_dir + documents_dir

This allows users to set `documents_dir: ""` to use the working directory for documents, which is useful for running wiki-go in different directories without changing config.
